This repository contains all code, notebooks, and outputs for a machine learning project that classifies images as real or AI-generated. Our goal is to surpass the reported human detection accuracy of ~62% using deep learning models trained on the [Defactify Image Dataset](https://huggingface.co/datasets/Rajarshi-Roy-research/Defactify_Image_Dataset).

---

## Repository Contents

This repository is organized into four top-level areas: `Data/` holds the dataset documentation, `Outputs/` holds all EDA visualizations, `Scripts/` holds all model training code and notebooks, and the root level contains the two primary analysis notebooks (`EDA_Proj3.ipynb` and `ensemble_training.ipynb`). The `Scripts/ViT/checkpoints/` folder stores the trained ViT model weights and evaluation results.

## Section 1: Software and Platform

### Software
- **Python 3.10+**
- **Jupyter Notebook** — used for EDA (`EDA_Proj3.ipynb`), CNN training (`CNNs.ipynb`), and ensemble training (`ensemble_training.ipynb`)

### Required Packages

Install all dependencies in one command:

```bash
pip install torch torchvision datasets scikit-learn matplotlib tqdm numpy pandas scipy scikit-image opencv-python Pillow timm transformers
```

| Package | Used In | Purpose |
|---|---|---|
| `torch` | All scripts & notebooks | Core deep learning framework for model training and inference |
| `torchvision` | ViT scripts, CNNs.ipynb | Image transforms, pre-trained CNN/ViT architectures |
| `datasets` | All notebooks, data_loader.py | Loading and saving the HuggingFace Defactify dataset |
| `scikit-learn` | evaluate.py, ensemble_training.ipynb | Metrics (accuracy, F1, ROC-AUC), logistic regression for ensemble |
| `matplotlib` | EDA, evaluate.py, CNNs.ipynb | All plots and visualizations |
| `tqdm` | All scripts & notebooks | Progress bars during training and data processing |
| `numpy` | All scripts & notebooks | Numerical operations and array handling |
| `pandas` | EDA, CNNs.ipynb, ensemble_training.ipynb | Tabular data manipulation |
| `scipy` | EDA_Proj3.ipynb | Kernel density estimation (`gaussian_kde`) for sharpness plots |
| `scikit-image` | EDA_Proj3.ipynb, CNNs.ipynb | Structural Similarity Index (SSIM) computation |
| `opencv-python` | EDA_Proj3.ipynb, CNNs.ipynb | Laplacian sharpness and image processing |
| `Pillow` | All notebooks | Image loading and format conversion |
| `timm` | ensemble_training.ipynb | Pre-trained EfficientNet model for ensemble |
| `transformers` | ensemble_training.ipynb | HuggingFace ViT (`ViTForImageClassification`) for ensemble |

### Platform
- Development and training were performed on **Windows** with a CUDA-compatible GPU (strongly recommended for practical training times).
- The code is compatible with **Mac** and **Linux** as well. If no GPU is available, the code automatically falls back to CPU via `device = torch.device("cuda" if torch.cuda.is_available() else "cpu")` in `config.py` — however, training will be significantly slower on CPU.

## Section 3: Instructions for Reproducing Results

Follow these steps in order to reproduce the full project results from scratch.

### Step 1: Clone the Repository and Set Up the Environment

Download or clone the repository to your local machine. It is recommended to use the included `.venv` virtual environment or create a fresh one:

```bash
python -m venv .venv
source .venv/bin/activate        # Mac/Linux
.venv\Scripts\activate           # Windows
```

Then install all required packages:

```bash
pip install torch torchvision datasets scikit-learn matplotlib tqdm numpy pandas scipy scikit-image opencv-python Pillow timm transformers
```

### Step 2: Download the Dataset

The dataset is the Defactify Image Dataset hosted on HuggingFace. Download and save it locally by running the following Python code (you can run this in a notebook cell or a standalone script):

```python
from datasets import load_dataset
dataset = load_dataset("Rajarshi-Roy-research/Defactify_Image_Dataset")
```

To replicate the exact subset used for ViT training (Real, SD3, and DALL-E 3 images only, corresponding to `Label_B` values 0, 3, and 4), filter and save the dataset:

```python
filtered = dataset.filter(lambda x: x["Label_B"] in [0, 3, 4])
filtered.save_to_disk("Scripts/ViT/data/defactify_filtered")
```

### Step 3: Run the EDA Notebook

Open `EDA_Proj3.ipynb` in Jupyter Notebook and run all cells from top to bottom. This will generate all plots saved to the `Outputs/` folder, including label distributions, RGB histograms, brightness and contrast comparisons, Laplacian sharpness distributions, 2D FFT frequency spectra for each AI model versus real images, and SSIM scores per model. No changes to the notebook are needed as long as the dataset was downloaded and accessible.

### Step 4: Verify the ViT Configuration

Open `Scripts/ViT/config.py` and confirm the paths and settings match your environment:

```python
CONFIG = {
    "save_path":       "Scripts/ViT/data/defactify_filtered",  # must match where you saved the dataset
    "checkpoint_path": "Scripts/ViT/checkpoints/vit_binary.pt",
    "batch_size":      32,    # reduce to 8–16 if running on CPU only
    "epochs":          20,
    "lr":              1e-5,
    "num_classes":     2,
    "freeze_backbone": False,
    "patience":        4,
    "num_workers":     2,
    "weight_decay":    0.01,
}
```

### Step 5: Train the ViT Model

Navigate into the `Scripts/ViT/` directory and run the training script:

```bash
cd Scripts/ViT
python train.py
```

During training you will see per-epoch output showing train loss, train accuracy, validation loss, validation accuracy, and validation accuracy broken down by source (Real, SD3, DALL-E 3). The best model checkpoint is automatically saved to `checkpoints/vit_binary.pt`. Training history is saved to `checkpoints/history.json`. Training will stop early if validation loss does not improve for 4 consecutive epochs.

### Step 6: Evaluate the ViT Model

Once training finishes, run the evaluation script from inside `Scripts/ViT/`:

```bash
python evaluate.py
```

This loads the best saved checkpoint and outputs the following to `checkpoints/`:
- Overall test metrics printed to the console: accuracy, precision, recall, F1-score, and ROC-AUC
- Per-source breakdown for Real, SD3, and DALL-E 3
- `confusion_matrix.png`
- `roc_curve.png`
- `per_source_metrics.png`
- `metrics.json` with all metrics saved for easy reference

### Step 7: (Optional) Run the CNN Models

Open `Scripts/Backup Scripts/CNNs.ipynb` in Jupyter Notebook and run all cells. This trains both a custom CNN from scratch and an EfficientNet model using transfer learning. Saved weights are written to `custom_cnn.pth` and `efficientnet.pth`, and a `comparison.png` plot is generated comparing the two models.

### Step 8: (Optional) Run the Ensemble Model

Open `ensemble_training.ipynb` at the root level in Jupyter Notebook and run all cells. This notebook combines EfficientNet and ViT predictions using logistic regression as a meta-learner, and evaluates the ensemble using the same accuracy, F1, and ROC-AUC metrics as the individual models.

### Expected Results

A successfully reproduced run should yield:
- **Test accuracy above 62%** — the reported human baseline
- **F1-score approaching 0.75** — our target threshold

Exact values may vary slightly due to random weight initialization and hardware differences, but overall trends and per-source rankings should be consistent across runs.
